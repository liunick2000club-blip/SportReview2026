# 📘 SportReview 2026 技术架构与文件说明书 (小白友好版)

这份文档旨在帮助非专业程序员快速理解 SportReview 2026 的工作原理、代码组织方式以及如何进行基础维护。

---

## 1. 核心技术大图 (The Big Picture)

这套应用是基于 **Next.js** 框架构建的全栈应用。我们可以把它想象成一个**餐厅**：

*   **前端 (Frontend / 门面)**：用户在浏览器里看到的网页，负责点菜和展示（使用 React + Tailwind CSS 构建）。
*   **后端 (Backend / 厨房)**：运行在服务器上，负责处理数据、校验身份、计算开支（Next.js API Routes）。
*   **数据库 (Database / 仓库)**：存储所有用户和运动记录的地方（使用 SQLite 数据库 + Prisma ORM 管理）。

## 2. 目录结构全解析

### 📂 `prisma/` —— 数据的“户口本”
这是应用最核心的数据定义层，决定了数据长什么样。
*   **`schema.prisma`**：**核心定义文件**。规定了 `User`（用户）和 `Activity`（运动记录）包含哪些信息。
*   **`migrations/`**：记录了数据库结构的变化历史。
*   **`prisma/dev.db`**：**真正的数据库文件**。所有的运动记录、账号信息都存在这个文件里。

### 📂 `src/app/` —— 路由与页面 (餐厅的各个房间)
Next.js 使用文件夹来代表网址路径：
*   **`page.tsx`**：**首页仪表盘**。展示饼图、柱状图和年度总览。
*   **`layout.tsx`**：全局外壳，包含字体设置和登录状态包裹器。
*   **`login/` & `register/`**：登录和注册页面。
*   **`new/` & `edit/`**：新增和修改记录的表单页面。
*   **`activities/`**：明细列表页，包含“导出 CSV”功能。
*   **`api/`**：**后端接口**（由前端代码在后台调用）。
    *   `api/auth/`：处理用户登录、注销和身份验证。
    *   `api/activities/`：负责运动数据的增删改查及 CSV 导出。
    *   `api/summary/`：负责年度汇总计算和年费摊销逻辑。
    *   `api/activities/sync/`：触发与 Strava 官网的同步。

### 📂 `src/lib/` —— 工具箱 (厨房的各种专用机器)
这里存放的是被各处调用的公共逻辑：
*   **`prisma.ts`**：数据库连接器，负责应用与数据库文件的通信。
*   **`strava.ts`**：专门负责与 Strava API 通信，处理授权与数据抓取。
*   **`dingtalk.ts`**：专门负责通过 Webhook 向您的钉钉群发送通知。
*   **`activity-parser.ts`**：解析钉钉消息或 CSV 的逻辑工具。

### 📂 `scripts/` —— 维护脚本 (扫地机器人的控制台)
这些文件通过命令行手动运行，用于辅助维护：
*   **`seed-admin.ts`**：初始化管理员、关联历史数据的脚本。
*   **`test-registration.ts`**：验证用户注册流程是否正常的测试工具。

## 3. 核心功能逻辑：数据是如何流动的？

### 场景 A：您点击“同步 Strava”
1.  **前端** (Dashboard) 发送请求给 `/api/activities/sync`。
2.  **后端接口** 获取当前登录用户的 Strava 配置信息。
3.  **lib/strava.ts** 携带密钥去问 Strava 官网：“给我 2026 年以来的运动数据”。
4.  拿到数据后，后端自动进行**查重**（避免重复录入），然后存入数据库。
5.  **前端** 收到成功反馈，刷新页面，图表即刻更新。

### 场景 B：费用摊销计算
在 `src/app/api/summary/route.ts` 中：
*   程序获取当前年份（2026）。
*   计算从 1 月 1 日到今天一共过了多少天。
*   按照 `3288 / 365 * 已过天数` 算出截止到此时此刻的“年卡已耗金额”。
*   累加数据库中所有的手动消费记录，得出最终的“运动总花费”。

## 4. 常见配置与环境变量 (`.env`)

`.env` 文件存放敏感信息，**请勿泄露**：
*   `DATABASE_URL`：数据库文件路径。
*   `NEXTAUTH_SECRET`：加密登录 Session 的随机密钥。
*   `STRAVA_CLIENT_ID/SECRET`：您的 Strava 开发平台凭证。
