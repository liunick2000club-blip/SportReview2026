# SportReview 应用开发路线图 (Roadmap)

本项目致力于打造一款高度个性化的运动记录与财务分析应用。以下是当前已确认的进阶开发方向：

## 🚀 第一阶段：数据可视化与趋势分析 (Trends)
*   **周/月跑步里程趋势：** 使用折线图展示每周或每月的累计跑步距离，直观查看耐力进步。
*   **攀岩频率分布图：** 优化柱状图，支持按月份筛选，对比不同月份的打卡热情。
*   **运动类型占比：** 增加饼图，展示跑步、骑行、攀岩、其他训练的时间/次数分配比例。

## 🧗‍♂️ 第二阶段：攀岩馆深度统计模块 (Gym Analysis)
*   **单馆详情页：** 点击馆名进入详情。
    *   统计在该馆的总次数、总时长（如果未来记录时长的话）、总花费。
    *   **性价比分析：** 计算该馆的单次平均成本（单次票 vs 年卡分摊）。
    *   **训练回顾：** 聚合展示在该馆留下的所有心得体会。

## 🎯 第三阶段：目标管理与激励系统 (Goals)
*   **月度目标设定：** 允许设置本月的跑步里程（如 50km）、攀岩次数（如 12次）。
*   **首页进度条：** 在仪表盘实时显示目标达成百分比。
*   **连续打卡记录：** 统计并展示最长连续运动天数，增加记录动力。

## 💾 第四阶段：数据主权与安全 (Data & Safety)
*   **一键导出功能：** 支持将数据库记录导出为 CSV 或 Excel 文件，方便备份。
*   **数据库快照：** 自动或手动备份 `dev.db` 镜像。
*   **多端适配优化：** 进一步优化移动端布局，确保在手机端录入和查看图表更加顺滑。

## 🛠 Docker 构建与部署 (NAS/服务器)
为了方便在群晖 NAS 或其他服务器上稳定运行，项目已完成 Docker 适配，以下是构建与部署步骤：

### 1. 本地构建镜像 (打包)
在项目根目录下执行以下命令，将代码打包成轻量级的 Docker 镜像：
```bash
docker build -t sport-review-2026:latest .
```

### 2. 导出与导入 (针对无公网环境)
如果 NAS 无法直接构建，可以先在本地导出镜像文件，再传输到 NAS 上：
*   **导出**：`docker save sport-review-2026:latest > sport-review-2026.tar`
*   **导入 (群晖)**：在 Docker 套件的“映像”页面，点击“新增” -> “从文件新增”并选择该 `.tar` 文件。

### 3. 使用 Docker Compose 启动
确保您的 `.env` 文件已配置正确（特别是 `NEXTAUTH_SECRET`），然后在代码目录下运行：
```bash
docker-compose up -d
```

### 4. 关键注意事项
*   **数据持久化**：必须将群晖的物理文件夹映射到容器内的 `/app/prisma/prisma`，否则重启容器会导致数据丢失。
*   **端口映射**：默认使用 `3000` 端口，可根据群晖实际情况在 `docker-compose.yml` 中修改。
